Вот пример пользовательского пути (user journey) для загрузки документов, ожидания подтверждения со стороны
администратора и проверки статуса подтверждения. Этот путь включает все основные шаги, которые пользователь должен
пройти.

### Пользовательский путь

#### 1. Регистрация и вход в систему

- **Регистрация**: Пользователь создает учетную запись на платформе, заполняя необходимые поля (имя, адрес электронной
  почты, пароль и т.д.).
- **Вход**: Пользователь входит в свою учетную запись, используя свои учетные данные.

#### 2. Переход к разделу загрузки документов

- **Навигация**: Пользователь переходит в раздел "Загрузка документов" через меню навигации на сайте.
- **Информация**: На странице загрузки документов пользователь видит инструкции о том, какие документы можно загрузить (
  например, паспорт, водительское удостоверение) и какие поля необходимо заполнить.

#### 3. Загрузка документов

- **Форма загрузки**: Пользователь заполняет форму, включая:
    - Выбор типа документа (например, паспорт, водительское удостоверение).
    - Ввод номера документа.
    - Ввод даты истечения срока действия документа.
    - Загрузка фотографий документа (можно загрузить несколько файлов).
    - Ввод описания документа (например, комментарии или дополнительные сведения).
- **Отправка**: После заполнения всех полей пользователь нажимает кнопку "Отправить" для загрузки документов.

#### 4. Ожидание подтверждения

- **Сообщение об успешной загрузке**: После отправки формы пользователь видит сообщение о том, что документы успешно
  загружены и находятся на рассмотрении.
- **Статус ожидания**: Пользователь может видеть статус своих загруженных документов как "На рассмотрении" в своем
  профиле.

#### 5. Уведомление о подтверждении

- **Проверка статуса**: Пользователь может периодически проверять статус своих документов в разделе "Мои документы".
- **Уведомление**: Как только администратор проверяет документы, пользователь получает уведомление (например, по
  электронной почте или через уведомление в приложении) о том, что его документы были подтверждены или отклонены.

#### 6. Просмотр статуса документов

- **Статус документов**: В разделе "Мои документы" пользователь видит список загруженных документов с их статусом:
    - **Подтверждено**: Если документы подтверждены, рядом с документом отображается зеленая галочка.
    - **Отклонено**: Если документы отклонены, отображается красный крестик и комментарий от администратора с причиной
      отклонения.
    - **На рассмотрении**: Если документы все еще находятся на рассмотрении, отображается статус "На рассмотрении".

#### 7. Действия после подтверждения

- **Обновление документов**: Если документы были отклонены, пользователь может загрузить новые или исправленные
  документы, следуя тем же шагам.
- **Обновление статуса**: Если документы подтверждены, пользователь может использовать их для доступа к дополнительным
  функциям платформы (например, для получения услуг, требующих подтверждения личности).

### Дополнительные функции

- **История загрузок**: Пользователь может видеть историю всех загруженных документов, включая даты загрузки и статусы.
- **Поддержка**: Если у пользователя есть вопросы или проблемы, он может обратиться в службу поддержки через форму
  обратной связи или чат.

Этот пользовательский путь обеспечивает четкое понимание процесса загрузки документов и взаимодействия с системой, а
также помогает пользователю оставаться в курсе статуса своих документов.
